Question-1: What is the difference between map,foreach,filter and find.

Map

The map method loops through each element in array and calls the provided function for each element. This method creates a new array and the original array is not modified.The result of the function is added to the new array in the same order as the original elements.

Foreach

forEach is used to execute the same code on every element in an array but does not change the array and it returns undefined.

Filter

The filter method in JavaScript creates a new array with the elements the provided condition. This method calls a provided function for each element in array and verifies the condition given in the provided function and passes only those elements that satisfy the given condition.

Find

This method returns first element of the array that satisfies the condition specified in the callback function.It does not create a new array.If no element passes the test, undefined is returned

Question-2: What is the difference between cookies,local storage and session storage.

Cookies

Cookies are small text files that are stored on the user's computer by the browser.Cookies have a size limit of 4KB. Cookies are vulnerable to Cross-Site Scripting (XSS) attacks, where malicious code can read or modify the contents of cookies.cookies are best used for storing small amounts of data that need to be sent to the server with every request.

Local Storage

Local Storage is a type of browser storage that allows you to store data on a user's browser without an expiration date.Local Storage has a much larger size limit than cookies, typically around 5-10MB depending on the browser.Local Storage is not vulnerable to CSRF attacks, but it is vulnerable to XSS attacks if the data stored in it is not properly sanitized.

Session Storage

Session Storage is similar to Local Storage, but the data is cleared when the user closes their browser or the tab.Session Storage has the same size limit as Local Storage, but the data is cleared when the session ends.Session Storage is also not vulnerable to CSRF attacks, but it is vulnerable to XSS attacks if the data stored in it is not properly sanitized.

Question-3: What is Event loop and how it works?

The Event Loop is a core concept in JavaScript that helps manage how code is executed in a non-blocking and efficient way.Event Loop is a mechanism that continuously checks the call stack and the task queue.When the call stack is empty, the Event Loop takes the first function from the task queue and pushes it onto the call stack to be executed.
how the Event Loop works:JavaScript code is executed in a single-threaded environment, which means that only one task can be executed at a time. If a function makes an asynchronous call the JavaScript engine does not wait for the call to complete. When the call stack is empty, the Event Loop checks the task queue for any pending tasks. This process is repeated continuously, resulting in a cycle that is known as the Event Loop.
